## Introduction and Project Goal
Welcome to the repository for the **Distributed Simultaneous Localization and Mapping (SLAM) Project** developed by **Team Zig-Zag**. 

Our primary objective is to implement, evaluate, and ultimately enhance a modern distributed SLAM system to meet the demands of contemporary robotic and autonomous systems. Specifically, we focus on **improved robustness in loss function design and loop closure mechanisms** within multi-robot environments.

This repository is built upon [DCL-SLAM](https://github.com/zhongshp/DCL-SLAM) as our core baseline framework, wrapping around robust LiDAR odometry systems like FAST-LIO2 and LIO-SAM.

## Key Features and Enhancements
Our project introduces critical robustness enhancements to the baseline distributed SLAM architecture:

- **Enhanced Loop Closure Validation:** Rewritten `loopclosure.cpp` to address "Catastrophic Loop Closure" (the "Snap" effect) caused by severe perceptual aliasing in repetitive indoor laboratory environments.
- **Improved Robust Loss Functions:** Integrating non-quadratic loss functions (Huber/Cauchy estimators) within the backend optimization to reject severe outliers generated by noisy inter-robot measurements.
- **Memory & Thread Safety Fixes:** Resolved OpenCV memory corruption and segmentation faults (`!fixedSize()` assertion failures) that occurred during high-load descriptor generation.
- **Consistent Pose Graph Optimization:** Corrected pose composition math, fixed missing iterator validations, and ensured consistent noise models for intra- versus inter-loop closures.

## System Architecture & Selection
After a thorough literature survey comparing systems like DiSCo-SLAM and visual SLAM alternatives, we selected **DCL-SLAM** due to its:
- **Lighting Invariance:** Operates entirely on LiDAR (Livox) and IMU, making it unaffected by darkness or severe glare.
- **Modular Frontend:** Highly flexible architecture that easily integrates with FAST-LIO2 or LIO-SAM.
- **Efficient Communication:** Transmits lightweight mathematical descriptors of point clouds instead of raw dense point cloud data.

## Prerequisites and Environment Setup
The project requires a standard ROS environment for distributed SLAM processing.

- **OS:** Ubuntu 18.04 or 20.04
- **Middleware:** ROS Melodic or ROS Noetic
- **Core Libraries:**
  - `CMake`, `Boost`, `PCL`, `Eigen`
  - `GTSAM` (Georgia Tech Smoothing and Mapping library)
  - `livox_ros_driver`
- **Build Tools:** `catkin_tools`, `wstool`

Install system dependencies:
```bash
sudo apt-get install cmake libboost-all-dev python-wstool python-catkin-tools
```

## Build Instructions
1. **Initialize the workspace:**
   ```bash
   mkdir -p ~/cslam_ws/src
   cd ~/cslam_ws
   catkin init
   catkin config --merge-devel
   catkin config --cmake-args -DCMAKE_BUILD_TYPE=Release
   ```
2. **Setup the repositories:** 
   Ensure `dcl_slam`, `dcl_fast_lio`, `dcl_lio_slam`, and `distributed_mapper` are inside your `src` directory.
3. **Build the packages:**
   ```bash
   catkin build dcl_lio_sam
   catkin build dcl_fast_lio
   ```
   *(Note: Ensure all dependencies like GTSAM are correctly linked during compilation).*

## Running the System
We have validated the system on public datasets (downloaded via Hugging Face) and custom laboratory data.

### Running with Datasets
1. Launch the distributed SLAM backend:
   ```bash
   roslaunch dcl_slam run.launch
   ```
2. Play the recorded rosbag:
   ```bash
   rosbag play <your-bag-path>.bag
   ```

### Real-World Lab Data (Phase III)
- **Agents:** 2 Independent UGVs (Robot $\alpha$ and Robot $\beta$)
- **Sensors:** Livox LiDAR, IMU
- **Environment:** Repetitive indoor corridors.
Ensure the correct configuration parameters are set in the `config/` directory tailored to the Livox sensor extrinsics and expected noise models to prevent perceptual aliasing.
